How to debug CoffeeScript code of this project

1. Install JetBrains Chrome Extension

There is jb.crx file in the root directory of this application, open Chrome's settings, select "Extension" pane and drag and drop jb.crx from file explorer to chrome. Then turn on the checkbox "Allow access to file URLs" and restart the extension (by disabling it and enabling it back).

2. Connect to Chrome

From Run Configuration list select "CoffeeScript Debug" and start it. RubyMine connects to Chrome, opens page to debug and Chrome should confirm it with yellow bar with text "JetBrains Chrome Extension is debugging this tab." on the top of the page.

3. Set breakpoint and fire the event

Open file debugging_example.js.coffee put breakpoint in the body of method "onButtonClick" and press the big green button on the page. Execution of CoffeeScript code should stop where breakpoint is located.

How to debug CoffeeScript code of your application

1. Make sure that JetBrains Chrome Extension is installed

Download file jb.crx here, open Chrome's settings, select "Extension" pane and drag and drop downloaded cbx from file explorer to chrome. Then check the "Allow access to file URLs" checkbox.

2. Turn on source-maps generation

In Gemfile add lines: 
gem "coffee-script-redux", :git => "git://github.com/netzpirat/ruby-coffee-script-redux.git"
gem "coffee-script-redux-debugger", :path => "~/Projects/GitHub/coffee-script-redux-debugger"

3. Create JavaScript Remote Debug run configuration

Open run configurations dialog. Select JavaScript Remote Debug.

  • Specify URL of the page you are going to debug.
  • Change browser to "Chrome"
  • Provide mapping to assets directory.

3. Connect to Chrome

Start Run Configuration you have just created and make sure that there is yellow bar with text "JetBrains Chrome Extension is debugging this tab." on the top of the page you are debugging.
